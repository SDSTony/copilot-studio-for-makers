# Lab 05 : 영수증 처리 에이전트

## 학습 목표

- 에이전트 플로우(Agent Flow)의 개념과 구성 방법 이해
- 프롬프트 도구(Prompt Action)를 활용하여 AI 기반 비즈니스 로직 구현
- OneDrive 커넥터를 사용하여 처리 결과를 파일로 저장
- 에이전트에 플로우 도구를 연결하여 end-to-end 자동화 구현

## 시나리오

- 임직원들은 출장 또는 업무 중 발생한 영수증을 사진으로 찍어 경비 처리 시스템에 업로드한다.
- 그런데 촬영된 파일명은 대부분 `IMG_20240315_143022.jpg` 같은 영수증의 내용물을 반영하지 않는 형태로 저장되어 있어, 실무자가 파일 업로드 시 어떤 영수증인지 파악하기 어렵다.
- 담당자는 파일을 일일이 열어 내용을 확인한 후 파일명을 수동으로 변경하는 번거로운 작업을 반복하고 있다.
- 본 실습에서는 영수증 이미지를 받아 그 내용(날짜, 금액, 사용처 등)을 AI가 분석하여 의미 있는 파일명으로 자동 변환한 뒤 OneDrive에 저장하는 에이전트를 만들어 본다.



## 지시사항

1. Copilot Studio에서 **"+ 빈 에이전트 만들기"** 버튼을 클릭하여 새로운 에이전트를 생성합니다. 에이전트 정보 입력 대화에서 다음과 같이 설정합니다:
   - **이름:** `영수증 처리 에이전트`
   - **스키마 이름:** `receiptAgent`

   그리고 **"만들기"** 버튼을 클릭합니다.
   ![1. 새 에이전트 만들기](imgs/receipt-agent/Slide1.JPG)
   

2. 에이전트가 생성되면 **"+ 도구 추가"** 버튼을 클릭합니다.

   ![2. 도구 탭에서 도구 추가](imgs/receipt-agent/Slide2.JPG)

3. **"에이전트 흐름"** (또는 **"워크플로"**)를 선택합니다.

   ![3. 에이전트 흐름 선택](imgs/receipt-agent/Slide3.JPG)

```{note}
에이전트 흐름(Agent Flow)은 정해진 순서대로 작업을 수행하는 결정론적(deterministic) 워크플로입니다. Lab04처럼 도구를 나열해두고 AI 오케스트레이터가 실행 순서를 판단하는 방식과 달리, 흐름은 "파일 수신 → AI 분석 → OneDrive 저장"처럼 단계가 항상 동일한 순서로 실행되어 안정적이고 예측 가능한 자동화가 필요한 경우에 적합합니다.
```

4. 에이전트 흐름 디자이너 화면이 열립니다. 첫 번째 노드 **에이전트가 호출을 받을 때**에서 **"+ 입력 추가"** 버튼을 클릭합니다.

   ![4. 입력 추가 버튼 클릭](imgs/receipt-agent/Slide4.JPG)

5. 사용자 입력 유형 선택 대화에서 **"파일"** 아이콘을 클릭합니다. 이 입력은 사용자가 영수증 이미지 파일을 업로드할 수 있도록 합니다.

   ![5. 파일 입력 유형 선택](imgs/receipt-agent/Slide5.JPG)

6. 파일 입력 파라미터의 이름과 설명을 설정합니다 (이러한 변수명은 한글이 지원될 때도 있으나, 웬만하면 영어를 쓰면 호환성이 좋습니다)
   - **이름:** `file content`
   - **설명:** `이름 변경 후 저장해야 하는 영수증`

   ![6. 파일 파라미터 설정](imgs/receipt-agent/Slide6.JPG)

7. 파일 입력 노드 아래의 **"+"** 버튼을 클릭합니다. 이는 파일을 받은 후 수행할 다음 작업을 추가합니다.

   ![7. 다음 작업 추가 버튼](imgs/receipt-agent/Slide7.JPG)

8. 작업 추가 대화에서 **AI 기능 > 프롬프트 실행**을 클릭합니다. 이 작업은 AI를 사용하여 영수증 이미지를 분석하고 파일명을 생성합니다.

   ![8. 프롬프트 실행 선택](imgs/receipt-agent/Slide8.JPG)

9. 프롬프트 실행 작업에서 **"새 연결 만들기"** 섹션을 채웁니다:
   - **연결 이름:** `receiptConnection` (또는 원하는 이름)
   - **인증 타입:** `OAuth` (기본 설정)
   
   **"로그인"** 버튼을 클릭하여 현재 Copilot Studio에 로그인된 Microsoft 계정으로 인증합니다.
```{note}
향후 다른 커넥터 작업을 추가할 때도 새 연결 만들기 화면이 뜬다면, 위와 같은 방법으로 연결을 잡아주면 됩니다.
```
   ![9. 프롬프트 연결 설정 및 로그인](imgs/receipt-agent/Slide9.JPG)

10. 인증 완료 후 **"프롬프트"** 드롭다운을 클릭합니다. **+ 새 사용자 지정 프롬프트**를 선택하여 새로운 프롬프트를 만듭니다.

   ![10. 프롬프트 템플릿 선택](imgs/receipt-agent/Slide10.JPG)

11. 프롬프트 명을 아래와 같이 설정하고, 모델도 변경합니다. 
   - **프롬프트 이름:** `이미지 파일 내용 기반으로 파일 이름 도출하기`
   - **모델:** `GPT-5 chat`

   ![11. 프롬프트 지시문 입력](imgs/receipt-agent/Slide11.JPG)

12. 프롬프트 설정에서 **입력 변수**를 추가해야 합니다. **/** 를 입력한 뒤 **이미지 또는 문서**를 클릭합니다. 이는 프롬프트가 영수증 이미지 파일을 입력으로 받도록 설정합니다.

   ![12. 입력 변수 추가 - 이미지 또는 문서](imgs/receipt-agent/Slide12.JPG)

13. 변수명을 `receiptImage`으로 설정합니다. **"샘플 데이터"** 섹션에서 **"이미지 또는 문서 업로드"**를 클릭하여 제공되는 실습 파일 중 테스트용 영수증 이미지를 업로드합니다.

   ![13. 테스트 이미지 업로드](imgs/receipt-agent/Slide13.JPG)

```{note}
여기서 올리는 샘플 이미지는 프롬프트를 설계하는 동안 결과를 미리 테스트해보기 위한 용도일 뿐입니다. 실제 실행 시에는 17단계에서 매핑하는, 사용자가 업로드한 영수증 파일이 입력으로 들어갑니다. 즉, 샘플은 개발 단계의 미리보기용 입니다. 에이전트가 실제 실행 될 때는 그 시점에 업로드된 파일을 가지고 활용하게 만들 예정입니다. 
```

14. 프롬프트 편집 화면에서 **지시문(Instructions)** 영역에 다음 내용을 입력합니다:
   
   ```
   위 파일을 참고해서, 파일 이름을 어떤 영수증인지 알 수 있도록 의미 있는 파일명으로 변환해주세요.
   
   파일명 규칙은 다음과 같아: YYYYMMDD_상점명_금액
   
   상점 이름은 한국어 또는 영어를 있는 그대로 사용
   
   결과는 JSON 형식으로 반환해주세요.
   
   키 이름으로 "filename"을 사용할 것
   ```
   
   이는 AI에게 영수증 이미지를 분석하고 규칙에 따라 파일명을 생성하도록 지시합니다.
   
   그리고 우측 상단의 **"출력: 텍스트"** 드롭다운에서 **JSON**을 선택합니다. 프롬프트의 출력을 JSON 형식으로 고정할 수 있습니다.

   ![14. 출력 형식을 JSON으로 설정](imgs/receipt-agent/Slide14.JPG)

15. **"테스트"** 버튼을 클릭하여 프롬프트를 실행합니다. 프롬프트가 업로드된 영수증 이미지를 분석하고 파일명을 생성합니다.

   ![15. 테스트 실행](imgs/receipt-agent/Slide15.JPG)

16. 테스트 실행 후 **"모델 응답"** 섹션에서 생성된 파일명을 확인합니다. JSON 형식으로 다음과 같은 형태로 반환됩니다:
   ```
   {
     "filename": "20260612_김공식식당_74000"
   }
   ```
   
   이제 이 결과를 OneDrive에 저장하는 작업을 추가해야 합니다. **"저장"** 버튼을 클릭하여 프롬프트를 저장합니다.

   ![16. 프롬프트 테스트 결과 확인 및 저장](imgs/receipt-agent/Slide16.JPG)

17. 에이전트 흐름 디자이너로 돌아갑니다. 번개 아이콘을 클릭한 뒤 **에이전트가 흐름을 호출할 때** 작업의 **file content contentBytes**를 선택합니다. 해당 변수에 분석 대상인 파일 콘텐츠가 존재합니다. 

   ![17. 프롬프트 실행 작업 자동 매핑](imgs/receipt-agent/Slide17.JPG)

18. 프롬프트 실행 노드 아래의 **"+"** 버튼을 클릭합니다. 작업 추가 대화에서 **"비즈니스용 OneDrive"**를 찾아 클릭합니다.

   ![18. OneDrive 커넥터 선택](imgs/receipt-agent/Slide18.JPG)

19. OneDrive 작업 목록에서 **파일 만들기**를 클릭합니다. 이 작업은 프롬프트에서 생성한 파일명을 활용해 영수증 이미지 파일을 OneDrive에 저장합니다.

   ![19. OneDrive 파일 만들기 작업 선택](imgs/receipt-agent/Slide19.JPG)

20. OneDrive 파일 만들기 작업에서 다음을 설정합니다:
   - **폴더 경로:** OneDrive에서 영수증 파일을 저장할 폴더 선택 (예: `/Accounts/CJ/Labs/Lab05/이름 변경된 파일`)

   ![20. OneDrive 폴더 경로 및 파일 이름 설정](imgs/receipt-agent/Slide20.JPG)

21. **파일 이름** 필드에서 동적 내용 버튼 **fx**을 클릭합니다. 그리고 아래의 수식을 입력한다.
   
   예를 들어, 다음과 유사한 식이 사용됩니다:
   ```
   concat(outputs('프롬프트_실행')?['body/responsev2/predictionOutput/structuredOutput/filename'], '.', 'jpg')
   ```

   ![21. 파일 이름을 동적 식으로 매핑](imgs/receipt-agent/Slide21.JPG)

```{note}
이 수식은 앞서 프롬프트 실행이 JSON으로 반환한 `filename` 값을 꺼내서(`.jpg` 확장자를 붙여) 파일 이름으로 사용하기 위한 것입니다. 14단계에서 출력을 JSON으로 고정해 둔 덕분에, 이처럼 정해진 경로(`.../structuredOutput/filename`)로 값을 안정적으로 참조할 수 있습니다.
```

22. **파일 내용** 필드에서 동적 내용 버튼 **번개 아이콘**을 클릭합니다. 입력 노드의 `file content contentBytes` 을 선택합니다. 

   ![22. 파일 내용을 동적으로 매핑](imgs/receipt-agent/Slide22.JPG)

23. 이제 **"Respond to the agent"** 노드를 설정하여 에이전트 흐름의 결과를 에이전트에 전달합니다. **"+ 출력 추가"** 버튼을 클릭합니다.

   ![23. 응답 노드에 출력 추가](imgs/receipt-agent/Slide23.JPG)

24. 응답 출력이 추가됩니다. **"output"** 변수명을 설정합니다.

   ![24. 응답 출력 매핑](imgs/receipt-agent/Slide24.JPG)

25. 번개 아이콘을 클릭한 뒤 **"파일 만들기"** 섹션에서 **"경로"** (또는 전체 결과)를 선택합니다. 

   ![25. 프롬프트 실행 출력값 선택](imgs/receipt-agent/Slide25.JPG)

26. 에이전트 흐름의 모든 노드와 연결이 완성되었습니다. 우측 상단의 **게시** 버튼을 클릭하여 에이전트 흐름을 저장합니다.

   ![26. 에이전트 흐름 저장](imgs/receipt-agent/Slide26.JPG)

27. 에이전트 흐름이 성공적으로 저장되었다는 메시지가 표시됩니다. 흐름에 머무르기를 클릭합니다. 

   ![27. 에이전트 흐름 저장 완료](imgs/receipt-agent/Slide27.JPG)

28. **"개요"** 탭으로 이동하여 흐름의 이름을 변경합니다. 
   - **이름:** `이미지 원드라이브 저장`

   ![28. 에이전트 개요 확인](imgs/receipt-agent/Slide28.JPG)

29. 에이전트의 **"도구"** 탭으로 이동합니다. 도구 추가 대화에서 생성한 에이전트 흐름 **"이미지 원드라이브 저장"**을 찾아 선택합니다. 이미 생성된 도구는 자동으로 나타납니다.

   ![29. 에이전트에 도구 추가](imgs/receipt-agent/Slide29.JPG)

30. 로딩이 완료되면 우측 하단의 **"추가 및 구성"** 버튼을 클릭하여 도구를 에이전트에 추가합니다.

   ![30. 도구 추가 및 구성](imgs/receipt-agent/Slide30.JPG)

31. 도구가 에이전트에 추가되고, **입력 추가** 버튼을 클릭하여 도구의 입력 파라미터(**contentBytes**, **name**)를 선택하여 추가합니다. 이를 통해 에이전트는 AI 판단하 적절한 값을 맵핑 합니다. 

   ![31. 도구 입력 설정 확인](imgs/receipt-agent/Slide31.JPG)

32. **완료** 섹션에서 **실행 후** 설정을 **생성형 AI로 응답 작성**으로 설정합니다. 그리고 **"저장"** 버튼을 클릭합니다. 

   ![32. 에이전트 지침 수정](imgs/receipt-agent/Slide32.JPG)

```{note}
**실행 후**를 **생성형 AI로 응답 작성**으로 설정하면, 흐름이 끝난 뒤 그 결과를 바탕으로 에이전트가 사용자에게 자연스러운 문장으로 결과를 안내합니다. 고정된 메시지 대신 상황에 맞는 응답을 생성하도록 하려는 것입니다.
```

33. 에이전트 설정이 저장되었습니다. **개요** 페이지로 돌아와서 **지침**을 아래와 같이 수정합니다. 

   ```
   파일이 첨부되면 /이미지 원드라이브 저장 을 사용해주세요. 
   ```

   ![33. 에이전트 설정 저장 완료](imgs/receipt-agent/Slide33.JPG)

34. **"테스트"** 탭에서 에이전트를 테스트합니다. 영수증 이미지 파일을 업로드하면:
   - 에이전트가 **에이전트 흐름 도구**를 실행합니다
   - 프롬프트가 영수증을 분석하여 의미 있는 파일명을 생성합니다 (예: `20260612_마트_74000`)
   - OneDrive에 자동으로 파일을 저장합니다
   
   실제 OneDrive 폴더에 접속하여 파일명이 올바르게 변환되어 저장되었는지 최종 확인합니다.

   ![34. 에이전트 테스트 및 최종 결과 확인](imgs/receipt-agent/Slide34.JPG) 

```{note}
테스트 패널에서 테스트 하는 것과, 게시 후 Teams 또는 M365 Copilot 채널에서 테스트 하는 것은 또 다른 결과물을 얻게 될 수 있다. 실제 에이전트 개발 시에는 테스트 패널에서 뿐만 아니라 게시 후 Teams 또는 M365 Copilot 채널에서도 원하는 작업들이 잘 수행되는지 확인 필요하다. 

가령, 이번 에이전트 같은 경우 게시해서 테스트 해보면, Teams 채널은 잘 되는데, M365 Copilot 채널은 잘 안될 수 있다.
```
