# Lab 06 : 이력서 분석 에이전트

## 학습 목표

- 토픽(Topic)을 사용하여 대화 흐름을 결정론적으로 설계하는 방법 이해
- **활동 발생(Activity Occurred)** 트리거를 활용하여 사용자 의도가 아닌 특정 이벤트(파일 첨부)로 토픽을 실행하는 방법 이해
- 시스템 변수(`System.Activity.Attachments`)를 읽고 별도 변수에 저장하는 방법 이해
- **목록을 통해 루프** 노드를 사용하여 여러 파일을 반복 처리하는 방법 이해
- 토픽 내에서 에이전트 플로우를 호출하고 Power Fx 수식으로 입력값을 전달하는 방법 이해

```{note}
에이전틱 하지는 않지만, Copilot Studio에서는 이러한 전통적인 로직 기반의 방법으로도 필요 시 프로세스 자동화를 진행해볼 수 있음을 확인해보는 것이 목표
```

## 시나리오

- 인사팀은 채용 과정에서 지원자들의 이력서를 다량 수신하며, 파일 형태(PDF)로 제출된 이력서에서 지원자 성함·경력·역량 등을 추출하여 별도 시트에 정리하는 작업(DB화 작업)을 반복하고 있다.
- 이력서를 여러 장 동시에 업로드하면 각 파일마다 자동으로 처리가 반복 실행되고, 사용자가 별도 명령어를 입력하지 않아도 파일 첨부 순간에 토픽이 자동으로 트리거되어 이력서 DB화를 자동으로 수행하는 결정론적(Deterministic) 구조를 구현한다.
