## 코드 인터프리터(Code Interpreter)란?

코드 인터프리터는 Copilot Studio 에이전트가 필요에 따라 **Python 코드를 직접 생성하고 실행**하여 파일을 처리하고 데이터를 계산하며 보고서를 생성할 수 있게 해주는 기능입니다. 대규모 언어 모델(LLM)이 스스로 숫자를 하나씩 읽어 추론(reasoning)으로 계산하는 방식과 달리, 코드 인터프리터는 실제 코드를 실행하므로 **정확하고 재현 가능한 결과**를 제공합니다.

```{image} imgs/code-interpreter/response-with-code-interpreter-python.png
:alt: 코드 인터프리터가 Python 코드를 실행하여 정확한 계산 결과를 반환하는 화면
:align: center
```

### 왜 코드 인터프리터가 필요한가?

LLM은 언어 생성에는 뛰어나지만 **정밀한 수치 계산에는 취약**합니다. 예를 들어 수십 개의 행이 있는 엑셀 파일에서 제품별 합계를 구하라고 요청하면, 모델은 숫자를 하나씩 읽으며 누적 합산을 시도합니다. 이 과정에서 값을 잘못 읽거나 중복 처리하는 등의 오류가 발생할 수 있어, 결과의 정확성을 보장하기 어렵습니다.

코드 인터프리터를 켜면 에이전트가 이러한 계산 작업을 **코드 실행으로 위임**하기 때문에, 데이터가 많고 계산이 복잡할수록 정확도가 크게 향상됩니다.

### 주요 활용 사례

- 엑셀/CSV 파일에서 **합계, 평균, 그룹별 집계** 등 정확한 수치 계산
- 업로드된 데이터 기반 **표, 차트, 보고서** 생성
- 복잡한 데이터 변환 및 **가공(전처리)**
- 파일 형식 변환 및 **결과 파일 생성**

### 제한 사항

- **프리미엄(Premium)** 기능이며 관리자가 사용을 설정하거나 관리할 수 있습니다.
- **프리뷰(Preview)** 기능으로, 프리뷰 사용 약관이 적용됩니다.
- 파일 업로드는 최대 **15MB**로 제한되며, 파일 형식 및 크기 제한이 적용됩니다.

## 실습: 영업 판매 데이터 합계 정확하게 계산하기

### 학습 목표

- 코드 인터프리터를 켜지 않았을 때 에이전트가 추론(reasoning)만으로 수치를 계산하며, 그 결과가 부정확할 수 있음을 확인한다.
- 코드 인터프리터를 활성화하여 에이전트가 Python 코드를 생성·실행하는 과정을 관찰하고, 계산 결과의 정확도가 향상되는 것을 비교한다.

### 시나리오

- 영업 담당자가 제품별 판매 금액이 정리된 엑셀 파일을 업로드하고, 각 제품별 판매 금액의 총 합계를 에이전트에게 질문한다.
- 코드 인터프리터를 켜기 전과 켠 후의 응답을 비교하여, 코드 인터프리터가 데이터 분석의 정확성에 미치는 영향을 직접 확인한다.

---

### 실습 단계

**1단계: Copilot Studio (<https://copilotstudio.microsoft.com/>)로 이동하여 새로운 에이전트를 하나 만든다.**

**2단계: 에이전트의 개요(개요) 화면에서 우측 상단의 테스트 패널을 연다. 계산이 필요한 질문(예: `각 제품별 판매금액 총 합계는 얼마에요`)을 입력하고, 클립 아이콘을 눌러 영업 판매 데이터 엑셀 파일(`영업 판매 데이터.xlsx`)을 첨부한 뒤 전송한다.**

```{image} imgs/code-interpreter/test-ask-sales-total-with-file.png
:alt: 테스트 패널에서 엑셀 파일을 첨부하고 판매금액 총 합계를 질문하는 화면
:align: center
```

**3단계: 에이전트의 응답을 확인한다. 상단 메뉴의 활동(활동)으로 이동하면 방금 나눈 대화 내역을 볼 수 있다. 이 시점에는 코드 인터프리터가 꺼져 있으므로, 에이전트는 추론(reasoning)만으로 계산한 결과를 제공한다.**

```{image} imgs/code-interpreter/response-without-code-interpreter.png
:alt: 코드 인터프리터 없이 추론만으로 계산한 판매금액 합계 결과
:align: center
```

**4단계: 응답의 추론(추론) 섹션을 펼쳐본다. 에이전트가 숫자를 한 줄씩 읽으며 수동으로 누적 합산을 시도하는 과정을 확인할 수 있다. 이 방식은 데이터가 많을수록 값을 잘못 읽거나 중복 처리할 가능성이 있어 결과의 정확성을 보장하기 어렵다.**

```{image} imgs/code-interpreter/reasoning-manual-calculation.png
:alt: 추론 섹션을 펼쳐 숫자를 하나씩 수동으로 계산하는 과정
:align: center
```

> ⚠️ 추론만으로 계산한 합계(예: `1,416,009,000원`)는 이후 코드 인터프리터로 계산한 결과와 다를 수 있습니다. 이는 LLM이 정밀한 수치 계산에 취약하다는 것을 보여줍니다.

**5단계: 이제 코드 인터프리터를 켠다. 개요 화면 우측 상단의 설정(설정) 버튼을 클릭한다.**

```{image} imgs/code-interpreter/open-settings.png
:alt: 개요 화면에서 설정 버튼 클릭
:align: center
```

**6단계: 설정 창의 좌측 메뉴에서 생성형 AI(생성형 AI)를 선택한다. 파일 처리 기능 섹션에서 코드 인터프리터(코드 인터프리터) 토글을 켜기로 전환한 뒤, 하단의 저장(저장) 버튼을 클릭한다.**

```{image} imgs/code-interpreter/enable-code-interpreter-toggle.png
:alt: 생성형 AI 설정에서 코드 인터프리터 토글을 켜고 저장
:align: center
```

> 💡 코드 인터프리터는 프리미엄·프리뷰 기능입니다. 토글이 보이지 않거나 비활성화되어 있다면 관리자에게 기능 사용 설정을 요청하세요.

**7단계: 다시 테스트 패널에서 동일한 엑셀 파일을 첨부하고 같은 질문(`각 제품별 판매금액 총 합계는 얼마에요`)을 입력하여 전송한다.**

**8단계: 활동으로 이동하여 응답을 확인한다. 이번에는 에이전트가 코드(프리뷰) 카드를 생성하며, `자세히 보기`를 클릭하면 실제로 실행된 Python 코드를 확인할 수 있다. 코드 실행을 통해 계산된 정확한 합계(예: `1,432,061,000원`)가 표 형태로 반환된다.**

```{image} imgs/code-interpreter/response-with-code-interpreter-python.png
:alt: 코드 인터프리터가 Python 코드를 실행하여 정확한 판매금액 합계를 반환한 결과
:align: center
```

**9단계: 4단계(코드 인터프리터 OFF)와 8단계(코드 인터프리터 ON)의 결과를 비교한다. 코드 인터프리터를 켰을 때 계산 과정이 코드 실행으로 대체되어 결과의 정확성과 신뢰성이 향상된 것을 확인할 수 있다.**

**10단계: 실제 정답을 직접 확인해본다. 업로드한 엑셀 파일을 열어 피벗 테이블이나 `SUMIF`, `SUM` 등의 엑셀 함수로 제품별 판매금액 합계를 계산하면 아래와 같은 값이 나온다.**

| 제품(Row Labels) | 판매금액 합계(Sum of 판매금액) |
|---|---|
| Azure AI | ₩434,397,000 |
| Copilot Studio | ₩433,625,000 |
| Microsoft 365 | ₩272,565,000 |
| Power Platform | ₩291,474,000 |
| **총 합계(Grand Total)** | **₩1,432,061,000** |

> ✅ 엑셀 함수로 계산한 총 합계 `₩1,432,061,000`은 **코드 인터프리터를 켰을 때(8단계)의 결과와 정확히 일치**합니다. 반면 코드 인터프리터를 끈 채 추론만으로 계산한 결과(4단계, 예: `₩1,416,009,000`)는 실제 값과 차이가 있습니다. 이를 통해 코드 인터프리터가 정확한 데이터 분석에 얼마나 중요한지 명확히 확인할 수 있습니다.

---

### 정리

- LLM은 언어 생성에는 강하지만 **정밀한 수치 계산에는 약하다**. 추론만으로 대량의 데이터를 합산하면 오류가 발생할 수 있다.
- 코드 인터프리터를 켜면 에이전트가 계산 작업을 **Python 코드 실행으로 위임**하여 정확하고 재현 가능한 결과를 제공한다.
- 데이터 분석, 집계, 보고서 생성 등 **정확성이 중요한 작업**에는 코드 인터프리터를 활성화하는 것이 좋다.

## 참고: 새로운 Copilot Studio에서는 기본으로 제공됩니다

위 실습은 **클래식(Classic) Copilot Studio** 환경을 기준으로, 코드 인터프리터를 직접 켜서 Python 코드 실행 기능을 활성화하는 과정을 다뤘습니다. 하지만 새롭게 개편된 **새 Copilot Studio(New Copilot Studio)** 경험에서 만드는 에이전트에는 이 기능이 **별도 설정 없이 기본적으로 활성화된 채로 제공**됩니다.

새 Copilot Studio는 **새로운 코딩 하네스(coding harness)와 CLI 계층(CLI layer)** 위에 구축된 새로운 에이전트 오케스트레이터를 사용합니다. 덕분에 새 환경에서 만든 에이전트는 다음과 같은 특징을 가집니다.

- **Python 코드 실행이 기본 내장**: 코딩 하네스와 CLI 계층을 기반으로, 에이전트가 필요에 따라 스스로 Python 코드를 생성·실행하여 계산·데이터 처리·파일 생성을 수행합니다. 앞선 실습처럼 코드 인터프리터를 수동으로 켤 필요가 없습니다.
- **더 강력한 지침 준수 및 장기(long-horizon) 작업 수행**: 재귀적(recursive) 작업 실행을 지원하여 복잡하고 동적인 문제를 여러 단계에 걸쳐 안정적으로 처리합니다.
- **대용량 콘텐츠 처리 및 리치 파일 출력**: 많은 양의 데이터를 처리하고 문서·데이터 파일을 생성하는 시나리오를 지원합니다.
